# Disciplina Padrões WEB (2026.1)

## Procedimento Metodológico 

Para conectar a teoria à prática de forma natural, criamos um contexto fictício neste repositório para ser utilizado 
com intuito de provocar discussões sobre conceitos e práticas de Padrões WEB com a turma **20261.1.19806.608.1M**. 
Para isso, a proposta é que estruturemos o ambiente de programação e as aulas com intuito de estabelecer períodos de **iteração** ( 7 dias, contemplando dois encontros), tempo necessário para darmos uma volta no ciclo de desenvolvimento de software, para promovermos **incrementos** no produto.  

Nesse período, os encontros serão utilizados para compreendermos o conteúdo da disciplina e o problema que queremos solucionar para desenvolvermos uma solução adequada 
(ver [Modelo de Desenvolvimento de Software Interativo e Incremental](https://pt.wikipedia.org/wiki/Desenvolvimento_iterativo_e_incremental)).

## Uma Breve Descrição do Contexto / Problema

A **FPenha Corporation**, uma holding multinacional com atuação consolidada no setor alimentício, contratou uma equipe para desenvolver uma solução digital estratégica. O desafio é modernizar a forma como a empresa disponibiliza seus cursos, dicas nutricionais e receitas, tornando o acesso à informação mais ágil e eficiente para o consumidor final.

Nosso objetivo é evitar que decisões de consumo sejam tomadas sem embasamento, oferecendo um portal estruturado, acessível e de alta performance.

Para atender aos requisitos dos nossos _stakeholders_, o sistema deve ser projetado com foco na experiência do usuário e na organização semântica das informações. O projeto abrange:

1. Arquitetura de Informação: Estruturação lógica de receitas e cursos.
2. Acessibilidade Digital: Garantir que o portal seja inclusivo, seguindo os padrões do W3C.
3. Performance: Garantir carregamento rápido em dispositivos móveis e desktops.
4. Semântica: Uso correto das tags HTML5 (<header>, <main>, <section>, <footer>) para SEO e legibilidade.
