
# Arquitetura de Layout Global e Alinhamento com Flexbox

Na vida real, os sites não deixam o texto esticar de uma ponta extrema da tela até a outra, pois isso cansa os olhos do usuário. Além disso, os cabeçalhos costumam organizar as marcas e os títulos horizontalmente (lado a lado).

Para atender à nova identidade visual solicitada pela FPenha Corporation, dividiremos nosso aprendizado em duas etapas estratégicas:

 - **O Contêiner Global**: Criar uma área limitada que abrace todo o conteúdo do site, centralizando-o em 80% da tela.
 - **O Cabeçalho Flexível**: Organizar o topo do site colocando o logotipo à esquerda e os títulos empilhados à direita.

## 1. Criando o Limite do Corpo: A importância da `<div>` de Conteúdo

O elemento `<body>` representa o navegador inteiro. Se quisermos limitar o conteúdo a apenas 80% da largura da tela, precisamos criar uma "caixa invisível" que embrulhe todo o nosso site (do topo ao rodapé).

Para isso, usamos a tag `<div>` (Digital Division). Ela é uma tag semântica genérica usada puramente para agrupar elementos e aplicar estilos de estrutura. Nós a chamaremos de uma classe personalizada: `.container`.

## 2. Alinhando Elementos Lado a Lado: O Poder do display: flex

Por padrão, vimos que títulos (`<h1>`) e imagens (`<img>`) tendem a se comportar de formas que nem sempre criam um layout horizontal amigável. Para colocar o logotipo do portal NossaReceita ao lado dos textos, utilizaremos a _tag_ semântica `<header>` combinada com o Flexbox.

Quando aplicamos `display: flex;` em uma caixa pai (o `<header>`), ela se transforma em um contêiner flexível, fazendo com que todos os seus filhos diretos (a imagem e a caixa de texto) fiquem instantaneamente lado a lado.

Para garantir que o título e o subtítulo continuem um embaixo do outro, nós os embrulhamos em uma `<div>` própria dentro do cabeçalho.

## 3. O Código HTML Estruturado e Semântico

## 4. O Código CSS Atualizado (Estilo.css)

Agora, vamos aplicar as propriedades físicas no arquivo `css/estilo.css` para fazer a mágica do layout acontecer:

``` css

/* Alvo: Qualquer elemento que assine a classe "caixa-contato" */
.caixa-contato {
    background-color: #f5f5f5; /* [cite: 653] */
    border: 2px solid #2e7d32;
    
    /* Afasta o conteúdo interno das bordas em 20 pixels, gerando respiro */
    padding: 20px; 
    
    /* Cria um espaço de 30 pixels abaixo do formulário, empurrando o footer */
    margin-bottom: 30px; 
}

/* Alvo Único: O elemento que possui o ID exclusivo "btn-enviar" */
#btn-enviar {
    background-color: #2e7d32; /* Fundo verde escuro institucional */
    color: #ffffff;            /* Texto branco para alto contraste */
    border: none;              /* Remove a borda padrão cinza do navegador */
    padding: 10px 20px;        /* 10px de preenchimento vertical e 20px nas laterais */
    cursor: pointer;           /* Altera a seta do mouse para o ícone de clique */
}
```

## 4. Workflow Git para esta Sprint

Alterações realizadas com sucesso? Como o ambiente de desenvolvimento é complexo e colaborativo, execute os comandos necessários no terminal para consolidar 
suas alterações sem gerar conflitos na equipe: 

 - Sincronizar (git pull): Garanta que sua máquina local possui os commits mais recentes do servidor.  
 - Preparar (git add .): Adicione as mudanças do HTML e do CSS para a área de espera (staging area).
 - Registrar (git commit -m "mensagem"): Crie seu ponto de restauração local. Exemplo: git commit -m "Aplicando os conceitos de classes, IDs e regras de padding no formulário".
 - Publicar (git push): Envie suas atualizações do computador local para o repositório remoto do GitHub.
