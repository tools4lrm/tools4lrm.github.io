
# Formatação e Semântica de Texto em HTML

Além de organizar a estrutura, um desenvolvedor front-end deve saber destacar informações de forma estratégica. A maneira como o texto é formatado impacta 
diretamente a Arquitetura da Informação e a Experiência do Usuário (UX). Em HTML, temos tags que mudam a aparência e tags que mudam o significado (importância) 
do texto para os **motores de busca** e **leitores de tela**.  

> **ATENÇÃO** : Todos os exemplos de código utilizados para representar os elementos poderão ser acessados e estudados, de como completo, [aqui](https://github.com/tools4lrm/tools4lrm.github.io/blob/068dec25f2589824c49f0b154f3c2c7ab38431c6/index.html).

## 1. Elementos de Estilo e Ênfase

Abaixo, as principais tags para edição e destaque de trechos específicos de conteúdo. 
### Tabela 1: Elementos HTML para Formatação de Texto
| Marcação | Função Técnica | Observação Semântica |
| --- | --- | --- | 
| `<b>` | Negrito (_Bold_) | Usado apenas para destaque visual, sem importância adicional. | 
| `<strong>` | Negrito de Importância | Indica que o texto é seriamente importante. É a tag recomendada para SEO. | 
| `<i>` | Itálico | Usado para termos técnicos, nomes científicos ou pensamentos. | 
| `<em>` | Ênfase | Indica que o texto deve ser enfatizado (geralmente lido com tom diferente). | 
| `<mark>` | Marca texto | Aplica uma cor de fundo (geralmente amarelo) para destacar termos de busca ou citações. | 
| `<small>` | Texto pequeno | Ideal para notas de rodapé, letras miúdas de contratos ou avisos de direitos autorais. | 
| `<del>` | Texto excluído | Exibe o texto riscado. Útil para mostrar preços antigos ou correções em artigos. | 
| `<ins>` | Texto inserido | Exibe o texto sublinhado. Frequentemente usado junto com o `<del>`. |
| `<sub>` | Texto subescrito | Posiciona o texto levemente abaixo da linha. | 
| `<sup>` | Texto sobrescrito | Posiciona o texto levemente acima da linha. |


## 2. Vamos aplicar estes elementos no Projeto 

No meu caso, para demonstrar ao cliente (FPenha Corporation) que dominamos a arte da formatação, vamos aplicar esses elementos no conteúdo que já temos. 

> **ATENÇÃO** : Vamos seguir o Fluxo de Trabalho Simples apresentado [aqui](aulas/27.03.md), pois trabalhar no desenvolvimento de software exige organização por
> ser uma atividade complexa e colaborativa. Se dois desenvolvedores mexem no mesmo arquivo sem ordem, o caos se instala.

### 2.1. Sincronizar (git pull)

Antes de digitar uma única linha de código, você deve trazer as atualizações do servidor para sua máquina local.
> O comando `git pull`, antes de iniciar qualquer mudança no seu repositório local, garante que você está trabalhando na versão mais recente do projeto e evitar conflitos com o trabalho de seus colegas que já foi enviado ao GitHub.

### 2.2. Modificar 

O **VS Code**, Ambiente de Desenvolvimento Integrado (IDE), é onde que desenvolvemos e evoluímos o nosso projeto/site. No meu caso, fiz as seguintes alterações/evoluções para
que as informações tivessem mais significado.

#### 2.2.1. Destacando no objetivo do site

Podemos usar `<strong>` e `<mark>` para garantir que o usuário não perca informações vitais sobre o projeto.  

``` html

...

        <p>
            Nosso <strong>objetivo</strong> é <mark>democratizar o conhecimento nutricional</mark>, 
            oferecendo conteúdos estruturados para que você aprenda 
            a fazer escolhas inteligentes, independentemente do seu 
            nível de experiência na cozinha. Ao explorar nosso portal, 
            você terá acesso a dicas exclusivas, cursos pensados para 
            o seu bem-estar e sugestões culinárias que respeitam o 
            seu ritmo e as necessidades do seu corpo, facilitando a 
            construção de hábitos duradouros que promovem mais 
            energia e longevidade para o seu dia a dia.
        </p>
    
...

```

#### 2.2.2. Evidenciando as alterações na Agenda de Cursos

Utilize o par `<del>` e `<ins>` para tornar mais explícito a mudança de datas e quantitativo de alunos.  

``` html

...

    <!-- o <table> é o elemento mais complexo do exemplo. -->
    <table border="1">
        <!-- <tr> (Table Row) cria a linha. Para pensarmos em colunas, temos que 
         antes pensarmos em linha, então, sempre terá uma linha -->
        <tr>
            <!-- <th> (Table Header) cria as celulas como cabeçalho. Dessa forma, os conteúdos dessas
             células serão renderizados em negrito e centralizados. -->
            <th>Dia</th>
            <th>Curso</th>
            <th>Nível</th>
            <th>Vagas</th>
        </tr>
        <tr>
            <!-- <td> (Table Data) preenche as células com o conteúdo informado. -->
            <td>De <del>15/05</del> para <ins>20/05</ins></td>
            <td>Saladas Tropicais</td>
            <td>Iniciante</td>
            <td>10</td>
        </tr>
        <tr>
            <td>22/05</td>
            <td>Molhos Naturais</td>
            <td>Intermediário</td>
            <td>De <del>08</del> para <ins>19</ins></td>
        </tr>
        <tr>
            <td>07/06</td>
            <td>Trabalhando com Ervas Finas</td>
            <td>Avançado</td>
            <td>05</td>
        </tr>
    </table>

 ...

```

#### 2.2.3. Ajuste no final da página 

Use `<small>` para direitos autorais e `<sub>` para informações técnicas nutricionais. 
Enquanto o itálico (`<i>`) ou ênfase (`<em>`) podem ser usados para termos em outros idiomas.

``` html

...

 <p>
        A ingestão diária recomendada de água é baseada na fórmula H<sub>2</sub>O 
        em conjunto com seu peso corporal. Quer saber mais sobre nutrição? Visite o 
        <!-- <a> (Âncora): É o link que conecta nossa página ao mundo exterior.
         - O atributo href é o destino (onde o usuário será conduzido ao clicar).
         - O atributo target="_blank" informa ao navegador que ele deverá abrir o 
         link em uma nova aba, garantindo que o usuário não abandone seu site ao 
         clicar para pesquisar em outro lugar. -->
        <a href="https://www.who.int" target="_blank">Portal da Organização Mundial da Saúde ( <i>World Health Organization</i> )</a>.
    </p>

    <!--O <footer> (rodapé) é um elemento semântico que define o encerramento de uma 
        seção ou da página inteira. Diferente de usar apenas um parágrafo no final da página,
        o <footer> comunica aos motores de busca (Google) e aos leitores de tela que:
            - O conteúdo principal terminou.
            - O que vem a seguir são informações de suporte (Copyright, 
            links de política de privacidade, redes sociais). -->
    <footer>
        <small>&copy; 2024 FPenha Corporation - Todos os direitos reservados.</small>
    </footer>
 
 ...

```

### 2.3. Preparar (git add .)
Concluído as alterações, precisarei notificar ao Git quais arquivos devem entrar no próximo ponto de evolução (commit).

<img width="3288" height="1884" alt="image" src="https://github.com/user-attachments/assets/ed39004c-30fd-44bd-8e15-d1b7f7df29c8" />

> O comando `git add .` informa ao git quais modificações eu quero confirmar. Seus arquivos saem da área de trabalho e entram na "Staging Area" (área de espera).


### 2.4. Registrar (git commit -m "mensagem")

Agora, irei confirmar as minhas alterações/evoluções no meu repositório local ( máquina local ) .

<img width="4556" height="1812" alt="image" src="https://github.com/user-attachments/assets/cbb097a8-bdd4-4f55-a63a-a38eb990077f" />


> O comando `git commit -m "Fazendo ajustes de formatações de textos"` cria um ponto de restauração.
> Se algo der errado no futuro, eu poderei voltar exatamente para este momento. A mensagem deve ser clara e descrever o que foi feito.

### 2.5. Publicar (git push)

Até agora, tudo está apenas no meu computador. Se meu HD quebrar, eu irei perde tudo. Além disso, como o ambiente de desenvolvimento é colaborativo, a minha equipe ainda não 
está visualizando as evoluções (nada do meu trabalho).

> Com o comando `git push`, os seus pontons de evolução/restauração local foram submetidos para o servidor do GitHub. Agora sim, a sua equipe de desenvolvimento
> está visualizando o seu trabalho.
