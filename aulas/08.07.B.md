
# Fluxo de Elementos (display), Larguras e Centralização

O portal NossaReceita já possui cores e espaçamentos básicos. No entanto, a diretoria da FPenha Corporation apontou um problema crítico de Experiência do Usuário (UX): os campos do formulário estão desalinhados, amontoados na mesma linha, e a caixa de contato ocupa a largura inteira da tela, o que dificulta a digitação em monitores grandes.

O cliente deseja que:

 - Cada campo do formulário (rótulos e caixas de texto) fique empilhado um abaixo do outro.
 - O formulário tenha um tamanho máximo restrito e fique centralizado elegantemente na página.

Para resolver essa demanda de negócio, vamos aprender sobre o comportamento de exibição dos elementos e o controle de dimensões no CSS.

## 1. O Comportamento dos Elementos: display: _block_

Por padrão, o navegador divide os elementos em dois grupos principais de comportamento:

 - **_Inline_ (Em linha)**: Elementos como `<label>`, `<input>` e `<a>` ocupam apenas o espaço do seu próprio conteúdo e tentam se posicionar um ao lado do outro. É por isso que o formulário está desalinhado.
 - **Block (Bloco)**: Elementos como `<h1>`, `<p>` e `<footer>` ocupam 100% da largura disponível e quebram uma linha automaticamente antes e depois de si mesmos, criando uma pilha **vertical**.

### A propriedade display

Para mudar esse comportamento, utilizamos a propriedade `display`. Ao aplicarmos `display: block` nas nossas _labels_, _inputs_ e _textareas_, forçamos o navegador a criar uma quebra de linha para cada um deles, empilhando o formulário perfeitamente.

## 2. Restringindo e Centralizando Caixas

Para evitar que o formulário estique infinitamente nas telas de computadores, usamos duas propriedades casadas:

 - **`max-width` (Largura Máxima)**: Define um limite de tamanho para a caixa (por exemplo, `500px`). Se a tela for menor que isso, a caixa encolhe; se for maior, ela trava no limite definido.
 - **`margin: 0 auto` (Margem Automática):** O valor `0` remove as margens superior e inferior, enquanto o valor auto diz ao navegador para calcular o espaço restante nas laterais e dividi-lo igualmente. Isso faz a caixa centralizar na tela instantaneamente.

## 3.O Código Atualizado

O nosso arquivo HTML permanece idêntico na estrutura, o que prova o poder do CSS: a manutenção é feita exclusivamente na folha de estilo, sem mexer na semântica da página.

> **ATENÇÃO** :  os exemplos de códigos contemplam apenas trechos, tenham o cuidado de analisarem o código como um todo.

``` html

    <h3>Fale Conosco</h3>
    <fieldset class="caixa-contato">
        <legend><strong>Contato com a Nutrição</strong></legend>
        <!-- Sobre os atributos action e method é o que separa o "desenhar o formulário" (Front-end) do "fazer o formulário funcionar 
             e mandar dados para o servidor" (Back-end).
             
             O atributo action (ação, em inglês) define para onde os dados do formulário serão enviados quando o usuário clicar no botão de envio (type="submit").

             O atributo method (método) define como os dados serão enviados para o endereço que definimos no action.
             -->
        <form action="#" method="POST">
            <label for="nome">Nome:</label>
            <input type="text" id="nome" name="nome" required>

            <label for="email">E-mail:</label>
            <input type="email" id="email" name="email" required>

            <label for="mensagem">Mensagem:</label>
            <textarea id="mensagem" name="mensagem" rows="8" required></textarea>

            <button type="submit" id="btn-enviar">Enviar</button>
        </form>
    </fieldset>

```

Como relação ao CSS, temos que compreender um pouco mais mais sobre seletores. Neste caso, seletores compostos. Olhando para o nosso arquivo `estilo.css`, você deve ter reparado nesta estrutura que parece um pouco mais complexa: 

``` css

/* --- SELETORES PERSONALIZADOS, BOX MODEL E DISPLAY --- */

.caixa-contato {
    background-color: #f5f5f5;
    border: 2px solid #2e7d32;
    padding: 20px; 
    
    /* MODIFICAÇÃO: Restringe o tamanho do formulário solicitado pelo cliente */
    max-width: 500px;
    
    /* MODIFICAÇÃO: Centraliza a caixa na tela e adiciona 30px de espaço abaixo */
    margin: 0 auto 30px auto; 
}

/* Alvo: Modificando as labels e campos internos do formulário para virarem blocos */
.caixa-contato label,
.caixa-contato input,
.caixa-contato textarea {
    display: block; /* Força cada elemento a ocupar sua própria linha */
    width: 100%;    /* Faz os inputs e textareas preencherem toda a largura interna da caixa */
    margin-bottom: 10px; /* Cria um pequeno espaçamento inferior entre os campos */
}

/* Ajuste específico para a label não colar no input */
.caixa-contato label {
    font-weight: bold;
    margin-bottom: 5px;
}

#btn-enviar {
    display: block;            /* Transforma o botão em bloco para aplicar margens */
    width: 100%;               /* Faz o botão ocupar toda a largura do formulário */
    background-color: #2e7d32; 
    color: #ffffff;            
    border: none;              
    padding: 12px;        
    cursor: pointer;           
    font-weight: bold;
}
```

Para um desenvolvedor profissional, escrever dessa forma economiza dezenas de linhas de código. Vamos destrinchar a lógica que o navegador usa para ler essa ordem, dividindo-a em duas regras de ouro:

### A Regra do Espaço (Seletor de Descendência / Filho)

Quando escrevemos `.caixa-contato label` (com um espaço entre eles), estamos dizendo ao navegador: "Procure a tag `<label>`, mas apenas se ela estiver guardada dentro de um elemento que tenha a classe `.caixa-contato`.

Isso é crucial para a segurança do projeto da FPenha Corporation. Se você tivesse outras label soltas pelo site em futuras páginas, elas não seriam afetadas por essa regra. Nós miramos com precisão cirúrgica apenas nas etiquetas do nosso formulário.

### A Regra da Vírgula (Agrupamento de Seletores)

A vírgula funciona literalmente como a palavra "e". Em vez de criarmos três blocos de código idênticos (um para a `label`, um para o `input` e um para a `textarea`), nós os separamos por vírgula. Programar também é evitar repetição de código (DRY - Don't Repeat Yourself).

O navegador lê a instrução assim:

"Aplique as regras abaixo para as `labels` da caixa de contato, E para os `inputs` da caixa de contato, E também para as `textareas` da caixa de contato."

## 4. Workflow Git para esta Sprint

Alterações realizadas com sucesso? Como o ambiente de desenvolvimento é complexo e colaborativo, execute os comandos necessários no terminal para consolidar 
suas alterações sem gerar conflitos na equipe: 

 - Sincronizar (git pull): Garanta que sua máquina local possui os commits mais recentes do servidor.  
 - Preparar (git add .): Adicione as mudanças do HTML e do CSS para a área de espera (staging area).
 - Registrar (git commit -m "mensagem"): Crie seu ponto de restauração local. Exemplo: git commit -m "Aplicando display block nos campos e centraliza o formulário com max-width".
 - Publicar (git push): Envie suas atualizações do computador local para o repositório remoto do GitHub.
