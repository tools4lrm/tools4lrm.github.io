# Introdução ao Git / GitHub

## 1. Preparando e Entendendo o Ambiente de Desenvolvimento

Desenvolver software não é apenas "digitar código". É uma atividade de engenharia que exige organização, colaboração e segurança. Essa atividade de desenvolvimento envolve diversas habilidades e ferramentas simultâneas:  

 - Escrita de Código: Lógica e sintaxe.  
 - Gestão de Dependências: Bibliotecas e frameworks externos.  
 - Versionamento: Saber quem mudou o quê, quando e por quê.  
 - Colaboração: Trabalhar no mesmo arquivo que outra pessoa sem apagar o que ela fez.  
 - Deploy (Implantação): Levar o código da sua máquina para o servidor real.  

 ---
 ## 2. O Ambiente de Desenvolvimento Moderno

 Para lidar com essa complexidade, dividimos as responsabilidades entre diferentes ferramentas que trabalham juntas.

### 2.1 Git (O Controle Local) 

O Git é uma ferramenta que proporciona um sistema de controle de versão dos artefatos do seu produto durante toda a vida útil.

 - Papel: Ele registra cada pequena alteração no seu código.
 - Onde fica: Na sua máquina local.
 - Função principal: Criar "pontos de restauração" (commits) para que você possa errar sem medo, sabendo que pode voltar no tempo se algo quebrar.

 ### 2.2 GitHub (A Plataforma de Colaboração)

O GitHub é uma plataforma online de hospedagem de código que oferece uma infraestrutura para desenvolvimento do seu produto. Dentro dessa infraestrutura oferecida, o Git é utilizado para controlar as evoluções durante o período útil do produto.  

 - Papel: É a "nuvem" do seu projeto e a rede social dos desenvolvedores.  
 - Onde fica: Nos servidores da Microsoft/GitHub (Acesso via navegador ou API).  
 - Função principal: Facilitar o trabalho em equipe através de ferramentas como Pull Requests, Issues e Actions (automação), além de servir como um portfólio público ou privado.

 ---
 ## 2.3 Fluxo de Trabalho Simples (Workflow)

Uma vez o ambiente esteja configurado com o repositório clonado, o desenvolvedor segue este ciclo lógico para manter a sanidade do projeto:

 | Passo | Ação | Comando | O que acontece? |
| -------- | ----- | ----------- | ------------------- |
| 1        | **Sincronizar**     | `git pull`     | Você baixa as novidades que outros colegas enviaram. |
| 2        | **Modificar**     | (_No VS Code_) | Você escreve seu código, cria bugs e conserta bugs. |
| 3        | **Preparar**     | `git add .` | Você separa as mudanças que estão prontas para serem salvas. | 
| 4        | **Registrar**     | `git commit -m "..."` | Você dá um nome àquela "foto" do seu código (Salva localmente). |
| 5        | **Publicar**     | `git push` | Você envia seus registros locais para o GitHub para que todos vejam. |

> **Dica de Boas Práticas**: Como já comentado, o ambiente de desenvolvimento de software é complexo e se faz necessário a colaboração. Então, quando estiver em um repositório envolvendo uma equipe, nunca comece a trabalhar sem dar um `git pull` antes. Isso evita os temidos **Conflitos de Merge**, que acontecem quando você e um colega mexem na mesma linha do mesmo arquivo ao mesmo tempo.
