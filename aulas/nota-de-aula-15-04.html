Aula 15/04: Primeiros Passos no Desenvolvimento Web

Assunto: Configuração de ambiente, Git e Estrutura Semântica

1. O "Pulo do Gato": Como entender o que estamos fazendo?

Analogia do Dia: Imagine que estamos construindo uma casa.

  • HTML (O Esqueleto): É a fundação, as vigas e as paredes. Ele diz onde fica a sala e onde fica a cozinha, mas sem acabamento.
  • Protocolo HTTP (O Carteiro): É o serviço que leva e traz as informações entre o seu computador e o servidor, garantindo que a "correspondência" (o site) chegue ao destino.

Lembre-se do mestre Steve Krug: Um site deve ser óbvio! A estrutura HTML serve para que o usuário e o navegador entendam o conteúdo sem precisar "pensar" demais.

2. Preparando o Canteiro de Obras (Passo a Passo)

  1. Abrir o VS Code: Nossa ferramenta de trabalho principal (o martelo e a trena).
  2. Git Clone: "Copiamos" o terreno onde vamos construir nosso projeto do servidor para o nosso computador.
  3. Navegação (cd): Entramos na pasta do projeto via terminal.
  4. Sincronização: Usamos git status e git fetch para ver se há novidades no projeto.
  5. Organização: Criamos um diretório específico para o projeto para não misturar os arquivos.

3. Nossa Primeira Estrutura (HTML5)

Baseado nos ensinamentos de Lucas Mazza e Jennifer Robbins, criamos o arquivo index.html. Este nome é importante: ele é a "porta de entrada" oficial de qualquer site.

<html>
    <head>
        <title>NossaReceita</title>
        <meta charset="utf-8" />
    </head>

    <body>
        <!-- O h1 é o título principal, o mais importante da página -->
        <h1>NossaReceita</h1>
        
        <!-- O h2 é um subtítulo que detalha o propósito -->
        <h2>
            Bem-vindo à NossaReceita. O seu portal de cursos, dicas 
            e receitas para uma vida saudável.
        </h2>
    </body>
</html>

4. Salvando o Progresso no "Cofre" (Git)

Após escrever o código, precisamos garantir que ele não se perca:

  • git add: Prepara os arquivos (coloca na caixa).
  • git commit: Registra a versão (fecha a caixa e coloca uma etiqueta).
  • git push: Envia para o servidor (guarda no cofre online).
