
# Interatividade e Captura de dados com Formulários HTML

Estruturamos o **portal NossaReceita** para que o usuário consuma informações (receitas, dicas e tabelas de cursos). Porém, a diretoria da FPenha Corporation 
trouxe uma nova necessidade de negócio: eles precisam reter esses usuários e abrir um canal direto de comunicação.  

O cliente deseja uma seção onde os visitantes possam tirar dúvidas sobre nutrição, enviar sugestões de receitas e demonstrar interesse nos cursos. 
Para resolver esse problema de negócio, nós, como desenvolvedores, utilizaremos os **Formulários HTML**.

## 1. Entendendo a Estrutura do Formulário

Um formulário é composto por um bloco principal e várias tags internas que mudam de comportamento dependendo de seus atributos e valores. 
Vamos analisar cada elemento solicitado pelo cliente:

### As Tags de Organização**
- `<fieldset>`: Funciona como uma caixa organizadora. Ele agrupa visual e semanticamente elementos relacionados dentro de um formulário.
- `<legend>`: É o título do `<fieldset>`. Ele fica encaixado na borda da caixa, deixando claro para o usuário (e para leitores de tela) o assunto daquele grupo de campos.
- `<form>`: É o contêiner principal. Tudo o que estiver dentro dele será empacotado e enviado quando o usuário clicar no botão de envio.

### As Tags de Entrada de Dados
 - `<label>`: É o rótulo do campo (o texto "Nome:", "E-mail:"). Ele possui o atributo crucial `for`, que deve apontar exatamente para o `id` do campo correspondente. Isso melhora a acessibilidade (se o usuário clicar no texto, o cursor vai direto para o campo de digitação).
 - `<input>`: É o campo de entrada de linha única. Seu comportamento muda drasticamente com o atributo `type`.
 - `<textarea>`: Diferente do `<input>`, este elemento permite que o usuário digite textos longos com múltiplas linhas (como o corpo de uma mensagem). Ele possui o atributo `rows` para definir sua altura inicial.
 - `<button>`: O gatilho que dispara a ação. Com o atributo `type="submit"`, ele avisa ao navegador que o formulário deve ser enviado.

## 2. Adicionando o Formulário ao Nosso Arquivo Principal

Para concluir a entrega do solicitado pela _FPenha Corporation_, vamos encaixar essa nova seção de interatividade logo acima do nosso rodapé (`<footer>`), 
mantendo a hierarquia e o fluxo lógico do documento:

``` html
...
    <h3>Fale Conosco</h3>
    <fieldset>
        <legend><strong>Contato com a Nutrição</strong></legend>
        <form action="#" method="POST">
            <label for="nome">Nome:</label>
            <input type="text" id="nome" name="nome" required>

            <label for="email">E-mail:</label>
            <input type="email" id="email" name="email" required>

            <label for="mensagem">Mensagem:</label>
            <textarea id="mensagem" name="mensagem" rows="8" required></textarea>

            <button type="submit">Enviar</button>
        </form>
    </fieldset>
...
```

### Sobre os atributos `action` e `method`

Podemos compreender esses atributos como sendo o que separa o "desenhar o formulário" (Front-end) do "fazer o formulário funcionar e mandar dados para o servidor" (Back-end).

- O atributo `action` (ação, em inglês) define para onde os dados do formulário serão enviados quando o usuário clicar no botão de envio (type="submit"). O valor desse atributo
deve ser a URL (o endereço) de um script no servidor (geralmente escrito em linguagens como PHP, Node.js, Java, Python ou C#) que estará preparado para receber e salvar essas informações no banco de dados.
O símbolo # indica que, ao enviar, a página apenas recarregará a si mesma, pois ainda não temos a URL do servidor do cliente pronta.
- O atributo `method` (método) define como os dados serão enviados para o endereço que definimos no action. Ele dita o protocolo de comunicação. Os dois valores mais importantes e utilizados no mercado são: `GET` e `POST`
