
# Expandindo Tabelas com `colspan` e `rowspan`

Até o momento, criamos tabelas onde cada linha possui exatamente a mesma quantidade de divisões (colunas). 
Na vida real, o cliente muitas vezes precisa de relatórios ou agendas com informações agrupadas, turnos unificados ou avisos que cruzam a tabela de um lado ao outro.  

Para resolver isso de forma elegante e semântica, o HTML nos fornece dois atributos fundamentais aplicados diretamente nas tags <td> ou <th>: colspan e rowspan.

> **ATENÇÃO** : Todos os exemplos de código utilizados para representar os elementos poderão ser acessados e estudados, de como completo, [aqui](https://github.com/tools4lrm/tools4lrm.github.io/blob/7639577e82657cd4ea8f6b74ae55e1e3803cb29f/index.html).

## 1. O Conceito de Mesclagem

Pense nos atributos como ordens de expansão que você dá para uma célula específica:
 - **colspan (Column Span / Expansão de Coluna)**: Faz uma única célula se esticar **horizontalmente** para ocupar o espaço de duas ou mais colunas.
 - **rowspan (Row Span / Expansão de Linha)**: Faz uma única célula se esticar **verticalmente** para ocupar o espaço de duas ou mais linhas (para baixo).

> **Regra de Ouro do Desenvolvedor**: Quando uma célula se expande, ela "empurra" ou ocupa o espaço das próximas células. Portanto, se você usou colspan="2",
> a linha atual precisará ter uma tag <td> a menos no código, pois a célula expandida já preencheu aquele espaço.

## 2. Nova Demanda do Cliente (Contexto de Negócio)

A diretoria da FPenha Corporation solicitou duas alterações urgentes na nossa Agenda de Cursos: (a) Os cursos de Saladas Tropicais e Molhos Naturais 
agora acontecerão no mesmo horário e terão o mesmo nível (Iniciante). Precisamos unificar essa informação verticalmente para poupar espaço. (b) Querem uma 
linha de aviso importante no final da tabela cruzando todas as colunas para informar sobre as inscrições.

## 3. O Código Reestruturado

Aqui está o arquivo `index.html` atualizado. Analise com atenção como as tags modificadas reduziram a quantidade total de elementos <td> nas linhas afetadas:

```html
...
   <h3>Agenda de Cursos de Maio</h3>

    <!-- o <table> é o elemento mais complexo do exemplo. -->
    <table border="1">
        <!-- <tr> (Table Row) cria a linha. Para pensarmos em colunas, temos que 
         antes pensarmos em linha, então, sempre terá uma linha -->
        <tr>
            <!-- <th> (Table Header) cria as celulas como cabeçalho. Dessa forma, os conteúdos dessas
             células serão renderizados em negrito e centralizados. -->
            <th>Dia</th>
            <th>Curso</th>
            <th>Nível</th>
            <th>Vagas</th>
        </tr>
        <tr>
            <!-- <td> (Table Data) preenche as células com o conteúdo informado. -->
            <td>De <del>15/05</del> para <ins>20/05</ins></td>
            <td>Saladas Tropicais</td>

            <!-- Expandindo/esticando verticalmente para ocupar o espaço de duas linhas -->
            <td rowspan="2">Iniciante</td>
            
            <td>10</td>
        </tr>
        <tr>
            <td>22/05</td>
            <td>Molhos Naturais</td>

            <!-- Removido o dado que será ocupado pela definição anterior. 
            <td> Avançado </td>       -->

            <td>De <del>08</del> para <ins>19</ins></td>
        </tr>
        <tr>
            <td>07/06</td>
            <td>Trabalhando com Ervas Finas</td>
            <td>Avançado</td>
            <td>05</td>
        </tr>

        <tr>
            <td colspan="4" align="center">
                <strong>Aviso:</strong> Inscrições <i>online</i> encerram 48 horas antes de cada curso!
            </td>
        </tr>

    </table>
...
```
