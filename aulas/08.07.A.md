## 1. O Conceito de Caixa: Margem e Preenchimento

Antes de mover ou estilizar elementos de forma avançada, todo desenvolvedor front-end deve compreender que o navegador interpreta cada elemento HTML como uma **caixa retangular**. 
Para controlar o tamanho e o respiro dessas caixas, usamos duas propriedades fundamentais:

 - _padding_ (Preenchimento Interno): **É o espaço** dentro da caixa, situado entre o conteúdo (como um texto) e a borda. Pense nele como a almofada de ar que protege o produto dentro de uma embalagem.
 - _margin_ (Margem Externa): **É o espaço** fora da caixa. Ele atua empurrando os outros elementos ao redor para longe, criando uma distância saudável entre caixas vizinhas.

O tempo todo estaremos lidando com **espaços dentro de uma área limitada pelo dispositivo que apresentará a informação**. Por ser uma área limitada, cada elemento influencia no posicionamento do outro elemento. Por isso, 
temos que sempre pensar no espaço, em como vamos preencher aquele espaço para depois combinar os elementos.

## 2. Dando Nome aos Elementos: Classes vs IDs

Para aplicar o _padding_ ou uma cor específica a apenas uma caixa na tela, precisamos de seletores mais cirúrgicos do que as _tags_ puras. É aqui que entram as **Class**es e os **ID**s.

### A) A Classe (_class_) — O Sobrenome dos Elementos

Uma classe serve para rotular / identificar elementos que compartilham o mesmo padrão de estilo. Você pode aplicar a mesma classe em vários elementos diferentes da página.

 - No HTML: Declaramos usando o atributo `class="nome-da-classe"`.
 - No CSS: Para chamar uma classe, iniciamos o seletor obrigatoriamente com um ponto (.) antes do nome.

### B) O ID (id) — O CPF do Elemento
O ID serve para identificar um elemento totalmente exclusivo na página. Um nome de ID nunca pode se repetir dentro do mesmo documento HTML.

 - No HTML: Declaramos usando o atributo id="nome-do-id".
 - No CSS: Para chamar um ID, iniciamos o seletor obrigatoriamente com uma hashtag (#) antes do nome.

## 3.Modificando o HTML do Portal NossaReceita

Vamos evoluir o código do nosso formulário de contato. Adicionaremos uma classe ao fieldset para gerenciar seu espaçamento interno e um ID exclusivo ao botão para garantir seu design único.

> **ATENÇÃO** :  os exemplos de códigos contemplam apenas trechos, tenham o cuidado de analisarem o código como um todo.

``` html

    <h3>Fale Conosco</h3>
    <fieldset class="caixa-contato">
        <legend><strong>Contato com a Nutrição</strong></legend>
        <!-- Sobre os atributos action e method é o que separa o "desenhar o formulário" (Front-end) do "fazer o formulário funcionar 
             e mandar dados para o servidor" (Back-end).
             
             O atributo action (ação, em inglês) define para onde os dados do formulário serão enviados quando o usuário clicar no botão de envio (type="submit").

             O atributo method (método) define como os dados serão enviados para o endereço que definimos no action.
             -->
        <form action="#" method="POST">
            <label for="nome">Nome:</label>
            <input type="text" id="nome" name="nome" required>

            <label for="email">E-mail:</label>
            <input type="email" id="email" name="email" required>

            <label for="mensagem">Mensagem:</label>
            <textarea id="mensagem" name="mensagem" rows="8" required></textarea>

            <button type="submit" id="btn-enviar">Enviar</button>
        </form>
    </fieldset>

```

``` css

/* Alvo: Qualquer elemento que assine a classe "caixa-contato" */
.caixa-contato {
    background-color: #f5f5f5; /* [cite: 653] */
    border: 2px solid #2e7d32;
    
    /* Afasta o conteúdo interno das bordas em 20 pixels, gerando respiro */
    padding: 20px; 
    
    /* Cria um espaço de 30 pixels abaixo do formulário, empurrando o footer */
    margin-bottom: 30px; 
}

/* Alvo Único: O elemento que possui o ID exclusivo "btn-enviar" */
#btn-enviar {
    background-color: #2e7d32; /* Fundo verde escuro institucional */
    color: #ffffff;            /* Texto branco para alto contraste */
    border: none;              /* Remove a borda padrão cinza do navegador */
    padding: 10px 20px;        /* 10px de preenchimento vertical e 20px nas laterais */
    cursor: pointer;           /* Altera a seta do mouse para o ícone de clique */
}
```

## 4. Workflow Git para esta Sprint

Alterações realizadas com sucesso? Como o ambiente de desenvolvimento é complexo e colaborativo, execute os comandos necessários no terminal para consolidar 
suas alterações sem gerar conflitos na equipe: 

 - Sincronizar (git pull): Garanta que sua máquina local possui os commits mais recentes do servidor.  
 - Preparar (git add .): Adicione as mudanças do HTML e do CSS para a área de espera (staging area).
 - Registrar (git commit -m "mensagem"): Crie seu ponto de restauração local. Exemplo: git commit -m "Aplicando os conceitos de classes, IDs e regras de padding no formulário".
 - Publicar (git push): Envie suas atualizações do computador local para o repositório remoto do GitHub.
